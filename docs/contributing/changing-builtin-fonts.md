# Changing a Built-in Reader Font

This guide explains how to replace one of the three built-in reader fonts (Bookerly, NotoSans, Literata) with a different font. All steps use Literata as the example being replaced.

## Prerequisites

- PlatformIO installed
- Python venv at `lib/EpdFont/scripts/.venv` (for font conversion)
- Ruby (for font ID generation)
- 4 static TTF files for the new font: Regular, Bold, Italic, BoldItalic

## Overview

A built-in font touches these locations:

| # | File | What to change |
|---|------|----------------|
| 1 | `lib/EpdFont/builtinFonts/source/<FontName>/` | Source TTF files |
| 2 | `lib/EpdFont/scripts/convert-builtin-fonts.sh` | Font name, sizes, source path |
| 3 | `lib/EpdFont/builtinFonts/<name>_<size>_<style>.h` | Generated headers (16 files for 4 sizes x 4 styles) |
| 4 | `lib/EpdFont/builtinFonts/all.h` | `#include` lines for generated headers |
| 5 | `lib/EpdFont/scripts/build-font-ids.sh` | Font ID generation (file references + `#define` names) |
| 6 | `src/fontIds.h` | Generated `#define` constants (auto-generated by script above) |
| 7 | `src/main.cpp` | Font variable declarations (~lines 95-118) and renderer registration (~lines 280-283) |
| 8 | `src/CrossPointSettings.h` | `FONT_FAMILY` enum value |
| 9 | `src/CrossPointSettings.cpp` | `getReaderFontId()` switch case and `getLineHeightMultiplier()` switch case |
| 10 | `lib/I18n/translations/*.yaml` | `STR_OPEN_DYSLEXIC` value (the UI display name) — all 12 language files |

## Step-by-step

### 1. Add font source files

Place 4 TTF files in `lib/EpdFont/builtinFonts/source/<NewFont>/`:
```
NewFont-Regular.ttf
NewFont-Bold.ttf
NewFont-Italic.ttf
NewFont-BoldItalic.ttf
```

### 2. Update conversion script

In `lib/EpdFont/scripts/convert-builtin-fonts.sh`:

```bash
# Change the variable name and sizes (currently 12 14 16 18, matching Bookerly/NotoSans)
LITERATA_FONT_SIZES=(12 14 16 18)
# becomes:
NEWFONT_FONT_SIZES=(12 14 16 18)

# Change the font_name prefix and font_path in the loop:
font_name="literata_${size}_..."
font_path="../builtinFonts/source/Literata/Literata-${style}.ttf"
# becomes:
font_name="newfont_${size}_..."
font_path="../builtinFonts/source/NewFont/NewFont-${style}.ttf"
```

### 3. Generate font headers

Delete old headers and generate new ones:
```bash
rm lib/EpdFont/builtinFonts/literata_*.h

cd lib/EpdFont/scripts
PYTHON=".venv/bin/python"
for size in 12 14 16 18; do
  for style in Regular Italic Bold BoldItalic; do
    font_name="newfont_${size}_$(echo $style | tr '[:upper:]' '[:lower:]')"
    $PYTHON fontconvert.py $font_name $size \
      ../builtinFonts/source/NewFont/NewFont-${style}.ttf \
      --2bit --compress > ../builtinFonts/${font_name}.h
  done
done
```

### 4. Update `all.h`

In `lib/EpdFont/builtinFonts/all.h`, replace all `literata` includes:
```cpp
// Replace all lines like:
#include <builtinFonts/literata_12_bold.h>
// with:
#include <builtinFonts/newfont_12_bold.h>
```

There are 16 include lines (4 sizes x 4 styles).

### 5. Update `build-font-ids.sh`

In `lib/EpdFont/scripts/build-font-ids.sh`, replace 4 blocks (one per size):
- Change `LITERATA_<size>_FONT_ID` to `NEWFONT_<size>_FONT_ID`
- Change `./literata_<size>_<style>.h` to `./newfont_<size>_<style>.h`

### 6. Regenerate `fontIds.h`

```bash
bash lib/EpdFont/scripts/build-font-ids.sh 2>/dev/null > src/fontIds.h
```

### 7. Update `main.cpp`

Two sections to change:

**Font declarations (~line 95-118):** Replace all `literata` variable names:
```cpp
// Pattern for each size (12, 14, 16, 18):
EpdFont newfont<Size>RegularFont(&newfont_<size>_regular);
EpdFont newfont<Size>BoldFont(&newfont_<size>_bold);
EpdFont newfont<Size>ItalicFont(&newfont_<size>_italic);
EpdFont newfont<Size>BoldItalicFont(&newfont_<size>_bolditalic);
EpdFontFamily newfont<Size>FontFamily(&newfont<Size>RegularFont, &newfont<Size>BoldFont,
                                      &newfont<Size>ItalicFont, &newfont<Size>BoldItalicFont);
```

**Renderer registration (~line 280-283):**
```cpp
renderer.insertFont(NEWFONT_12_FONT_ID, newfont12FontFamily);
renderer.insertFont(NEWFONT_14_FONT_ID, newfont14FontFamily);
renderer.insertFont(NEWFONT_16_FONT_ID, newfont16FontFamily);
renderer.insertFont(NEWFONT_18_FONT_ID, newfont18FontFamily);
```

### 8. Update `CrossPointSettings.h`

In the `FONT_FAMILY` enum (~line 83):
```cpp
enum FONT_FAMILY { BOOKERLY = 0, NOTOSANS = 1, LITERATA = 2, FONT_FAMILY_COUNT };
// becomes:
enum FONT_FAMILY { BOOKERLY = 0, NOTOSANS = 1, NEWFONT = 2, FONT_FAMILY_COUNT };
```

**Important:** Keep the numeric value `= 2` the same to preserve saved user settings.

### 9. Update `CrossPointSettings.cpp`

Two switch statements reference the font:

**`getReaderFontId()` (~line 322):**
```cpp
case NEWFONT:
  switch (fontSize) {
    case SMALL:          return NEWFONT_12_FONT_ID;
    case MEDIUM: default: return NEWFONT_14_FONT_ID;
    case LARGE:          return NEWFONT_16_FONT_ID;
    case EXTRA_LARGE:    return NEWFONT_18_FONT_ID;
  }
```

**`getLineHeightMultiplier()` (~line 250):**
```cpp
case NEWFONT:
  switch (lineSpacing) {
    case TIGHT:          return 0.90f;
    case NORMAL: default: return 0.95f;
    case WIDE:           return 1.0f;
  }
```

### 10. Update translation files

In **all 12 files** under `lib/I18n/translations/*.yaml`, change line 175:
```yaml
STR_OPEN_DYSLEXIC: "Literata"
# becomes:
STR_OPEN_DYSLEXIC: "New Font Name"
```

Files: catalan, czech, english, french, german, portuguese, romanian, russian, spanish, swedish, ukrainian, vietnamese.

Quick command:
```bash
for f in lib/I18n/translations/*.yaml; do
  sed -i '' 's/STR_OPEN_DYSLEXIC: "Literata"/STR_OPEN_DYSLEXIC: "New Font Name"/' "$f"
done
```

### 11. Build and test

```bash
pio run
```

## Quick reference: find-and-replace summary

If your new font's internal name is `newfont` (lowercase, no spaces):

| Find | Replace | Scope |
|------|---------|-------|
| `literata` | `newfont` | all `.cpp`, `.h`, `.sh` files listed above |
| `LITERATA` | `NEWFONT` | all `.cpp`, `.h`, `.sh` files listed above |
| `Literata/Literata` | `NewFont/NewFont` | `convert-builtin-fonts.sh` (source path) |
| `"Literata"` | `"Display Name"` | `lib/I18n/translations/*.yaml` |

## Notes

- Font sizes (12, 14, 16, 18) should match Bookerly/NotoSans for consistent sizing across font families
- The `STR_OPEN_DYSLEXIC` key name is a legacy from when this slot was OpenDyslexic — it's just a string key, the actual display name comes from the YAML value
- Flash impact: each font family (4 sizes x 4 styles) uses ~0.8-1.1 MB depending on glyph complexity
- Current flash budget: ~6.55 MB total, ~1.5 MB remaining after 3 fonts
